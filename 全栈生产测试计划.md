# 知识库系统 — 全栈生产测试计划

> 版本：1.0 | 日期：2026-06-16 | 目标配置：`BACKEND=postgres` + `MILVUS_ENABLED=true` + `MINIO_ENABLED=true`

---

## 一、测试目标与范围

### 1.1 目标

验证知识库系统在 **全栈生产配置**（PostgreSQL + Milvus + MinIO + 火山引擎 LLM/Embedding）下的功能完整性、数据一致性、性能指标、容错能力和前后端集成正确性。

### 1.2 范围边界

| 维度 | 包含 | 不包含 |
|------|------|--------|
| 后端功能 | 9 个 API 端点全部 | — |
| 解析器 | 6 种格式（md/docx/html/pdf/pptx/xlsx） | 未来新增格式 |
| 索引引擎 | Milvus Dense + Sparse Hybrid | 纯内存模式回归（已有覆盖） |
| 存储 | PostgreSQL + MinIO | 本地文件存储 |
| 前端 | 6 个页面 + Hash 路由 + API 调用 | 浏览器兼容性矩阵 |
| LLM 链路 | 语义抽取、查询改写、Rerank、视觉理解 | LLM 模型本身精度评测 |
| 增量更新 | 乐观锁 + 级联子文档 + 旧 chunk 淘汰 | — |
| 去重 | SHA256 哈希去重 | — |

### 1.3 测试环境

| 组件 | 版本/规格 | 说明 |
|------|-----------|------|
| PostgreSQL | 16+ | 通过 docker-compose 启动 |
| Milvus | 2.4+ | 通过 docker-compose 启动，含 etcd |
| MinIO | latest | 通过 docker-compose 启动 |
| 火山引擎 Ark | 生产 API | 需有效 `VOLCENGINE_API_KEY` |
| Python | 3.10+ | 项目运行环境 |
| 浏览器 | Chrome/Firefox 最新版 | 前端手工 / E2E 测试 |

## 二、测试层次与策略

```
                    ┌─────────────────┐
                    │   手工验收测试    │  ← 全链路业务场景
                    ├─────────────────┤
                    │   E2E 端到端     │  ← 前端 → 后端 → 所有服务
                    ├─────────────────┤
                    │   集成测试       │  ← 跨模块边界、外部服务交互
                    ├─────────────────┤
                    │   单元测试       │  ← 纯逻辑、Mock 外部依赖
                    └─────────────────┘
```

| 层次 | 工具 | 覆盖率目标 | 运行频率 |
|------|------|-----------|----------|
| **单元测试** | pytest + unittest.mock | ≥ 80% | 每次 commit |
| **集成测试** | pytest + docker-compose + 真实服务 | 核心链路 100% | 每次 PR |
| **E2E 测试** | pytest + requests / Playwright | 关键用户旅程 | 每次发布 |
| **手工验收** | 浏览器 + curl | 全场景 | 发布前 |

## 十五、风险与缓解

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
