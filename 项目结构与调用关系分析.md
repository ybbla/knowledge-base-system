## 三、核心调用链路（数据流）

### 3.1 入库链路：`upload → ingest → parse → extract → index`

```
┌──────────────────────────────────────────────────────────────────────────┐
│                          入库数据流                                       │
├──────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  [客户端]                                                                │
│     │                                                                    │
│     ▼                                                                    │
│  POST /upload                    POST /ingest                            │
│  ┌──────────────┐               ┌──────────────────┐                    │
│  │ upload.py    │               │ ingest.py        │                    │
│  │              │               │                  │                    │
│  │ 1. SHA256    │  source_uri   │ 1. 去重检查       │                    │
│  │    hash 计算  │──────────────▶│    (find_by_hash) │                    │
│  │ 2. MinIO/    │  source_hash  │ 2. 新建/更新分支   │                    │
│  │    本地落盘   │               │ 3. 提交异步任务    │                    │
│  └──────────────┘               └───────┬──────────┘                    │
│                                         │                                │
│                            IngestionPipeline.submit()                    │
│                                         │                                │
│                    ┌────────────────────┼────────────────────┐           │
│                    ▼                    ▼                     ▼          │
│              ┌──────────┐      ┌──────────────┐      ┌─────────────┐    │
│              │ Parser   │      │ Semantic     │      │ Indexing    │    │
│              │ Registry │      │ Extractor    │      │             │    │
│              │          │      │              │      │             │    │
│              │ .md  →   │      │ 窗口切分      │      │ Embedding   │    │
│              │ .docx→   │      │ (h2边界+     │      │ (火山1024d) │    │
│              │ .html→   │─────▶│  token限制)  │─────▶│             │    │
│              │ .pdf →   │      │              │      │ Dense →     │    │
│              │ .pptx→   │      │ LLM 抽取      │      │ Milvus/     │    │
│              │ .xlsx→   │      │ → Chunks     │      │ Memory      │    │
│              └──────────┘      └──────────────┘      │             │    │
│                    │                    │             │ Sparse →    │    │
│                    ▼                    ▼             │ Milvus/     │    │
│              ┌──────────┐      ┌──────────────┐      │ Memory      │    │
│              │ Asset    │      │ Chunk Store  │      └─────────────┘    │
│              │ Processor│      │ (PG/Memory)  │                         │
│              │          │      └──────────────┘                         │
│              │ 图片视觉  │                                                │
│              │ 理解+去重 │                                                │
│              │ 上传MinIO│                                                │
│              └──────────┘                                                │
└──────────────────────────────────────────────────────────────────────────┘
```

**步骤详解**：

| 步骤 | 组件 | 输入 | 输出 |
|------|------|------|------|
| ① 上传 | `upload.py` | 文件二进制 | `source_uri`, `source_hash` |
| ② 去重检查 | `DocumentRepository.find_by_hash()` | source_hash | 命中→duplicate；未命中→继续 |
| ③ 解析 | `ParserRegistry.get(type).parse(doc)` | Document | `ParsedElement[]` + `Asset[]` |
| ④ 递归加载 | `RecursiveLoader.load_embedded()` | elements | 子文档 elements |
| ⑤ 资源处理 | `process_image()` / `process_video()` | Asset | 视觉描述 + MinIO 上传 |
| ⑥ 语义抽取 | `SemanticExtractor.extract()` | elements + assets | `KnowledgeChunk[]` |
| ⑦ 索引写入 | `_index_chunks()` | chunks | Embedding → Milvus/Memory 双写 |
| ⑧ 状态更新 | `DocumentRepository.update()` | doc | status=active |

### 3.2 检索链路：`search → rewrite → dual retrieval → RRF → rerank`

```
┌──────────────────────────────────────────────────────────────────────────┐
│                          检索数据流                                       │
├──────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  POST /search                                                            │
│  ┌──────────────────┐                                                    │
│  │ search.py        │                                                    │
│  │                  │                                                    │
│  │ RetrievalPipeline│                                                    │
│  └───────┬──────────┘                                                    │
│          │                                                               │
│          ▼                                                               │
│  ┌──────────────────┐                                                    │
│  │ 1. Query Rewrite │  LLM 改写查询 → rewritten_query + keywords        │
│  └───────┬──────────┘                                                    │
│          │                                                               │
│          ├────────────────────┬────────────────────┐                     │
│          ▼                    ▼                    │                     │
│  ┌──────────────┐    ┌──────────────┐             │                     │
│  │ Vector Search│    │ BM25 Search  │             │                     │
│  │ (dense)      │    │ (sparse)     │             │                     │
│  │              │    │              │             │                     │
│  │ Milvus:      │    │ Milvus:      │             │                     │
│  │  COSINE      │    │  IP + TF-IDF │             │                     │
│  │  top_k=50    │    │  top_k=50    │             │                     │
│  │              │    │              │             │                     │
│  │ Memory:      │    │ Memory:      │             │                     │
│  │  numpy cosine│    │  rank_bm25   │             │                     │
│  └──────┬───────┘    └──────┬───────┘             │                     │
│         │                   │                     │                     │
│         ▼                   ▼                     │                     │
│  ┌──────────────────────────────────────┐         │                     │
│  │ 2. RRF Fusion (k=60)                 │         │                     │
│  │    Milvus: hybrid_search + RRFRanker │         │                     │
│  │    Memory: 应用层 RRF                │         │                     │
│  └──────────────┬───────────────────────┘         │                     │
│                 │                                 │                     │
│                 ▼                                 │                     │
│  ┌──────────────────────────┐                    │                     │
│  │ 3. LLM Rerank            │                    │                     │
│  │    重排序 top_k=20 → 5   │                    │                     │
│  └──────────────┬───────────┘                    │                     │
│                 │                                 │                     │
│                 ▼                                 │                     │
│  ┌──────────────────────────┐                    │                     │
│  │ SearchResult             │                    │                     │
│  │  • score_components      │                    │                     │
│  │  • source_refs           │                    │                     │
│  │  • asset_refs (含预签名URL)│                   │                     │
│  └──────────────────────────┘                    │                     │
└──────────────────────────────────────────────────────────────────────────┘
```

### 3.3 增量更新链路

```
POST /ingest (doc_id 有值)
        │
        ▼
┌─────────────────────────┐
│ ingest.py               │
│ 1. 查已有 Document       │
│ 2. hash 相同 → no_change │
│ 3. hash 不同 → 更新流程   │
└────────┬────────────────┘
         │
         ▼
┌──────────────────────────────┐
│ IngestionPipeline._run_update│
│                              │
│ 1. 乐观锁 version += 1       │
│ 2. 级联查找子文档             │
│ 3. 重新解析 → 新 elements    │
│ 4. 重新抽取 → 新 chunks      │
│ 5. 新 chunks 写入索引         │
│ 6. 旧 chunks → superseded    │  ← PG + Milvus 双标记
│ 7. 更新文档状态               │
└──────────────────────────────┘
```

## 五、后端可切换架构

系统通过环境变量实现 **三套后端组合**：

| 配置 | `BACKEND` | `MILVUS_ENABLED` | `MINIO_ENABLED` | 索引 | 存储 | 数据库 |
|------|-----------|------------------|-----------------|------|------|--------|
| **纯内存**（默认） | `memory` | `false` | `false` | MemoryVector + MemoryBM25 | MemoryAssetStore | 无 |
| **PG + 内存索引** | `postgres` | `false` | `false` | MemoryVector + MemoryBM25 | PgAssetStore | PostgreSQL |
| **全栈生产** | `postgres` | `true` | `true` | MilvusVector + MilvusSparse | MinioAssetStore | PostgreSQL |

切换逻辑集中在 [app/core/deps.py](knowledge_base_system/app/core/deps.py) 的模块级代码中，启动时根据配置一次性组装。

## 六、数据模型层次

```
Document (文档)
  │
  ├── ParsedElement[] (解析元素)
  │     ├── element_type: title | paragraph | table | image | video | ...
  │     ├── asset_ids → Asset[]
  │     └── embedded_doc_id → 子 Document (递归)
  │
  ├── KnowledgeChunk[] (知识块 — LLM 生成)
  │     ├── knowledge_type: declarative | relational | procedural
  │     ├── status: active | superseded
  │     ├── index_status: pending → indexing → indexed | failed
  │     ├── source_refs → SourceRef[] (溯源)
  │     └── asset_refs → AssetRef[] (资源引用)
  │
  └── Asset[] (资源)
        ├── asset_type: image | video | ...
        ├── status: pending → ready | failed | skipped
        ├── extracted_text (视觉理解结果)
        └── storage_uri (MinIO/本地路径)
```

## 十二、异常体系

```
KnowledgeBaseError (基类)
├── ParseError           — 文档解析失败
├── LLMError             — LLM 调用失败（超过重试次数）
├── IndexingError        — 索引操作失败
├── RetrievalError       — 检索管道失败
├── DuplicateDocumentError — 尝试创建已存在的文档
├── VersionConflictError — 乐观锁版本冲突
└── DocumentNotFoundError — 文档不存在
```

## 总结

这是一个设计良好的**多层可切换架构**的知识库系统。核心流程是：

```
上传 → 解析 → LLM语义抽取 → 双路索引 → 混合检索
```

通过 [deps.py](knowledge_base_system/app/core/deps.py) 的 DI 容器实现 memory/postgres/milvus/minio 的灵活组合。当前已支持：

- **6 种文档格式**：Markdown、DOCX、HTML、PDF、PPTX、XLSX
- **多模态视觉理解**：图片描述 + 视频总结
- **文档去重与增量更新**：SHA256 哈希去重 + 乐观锁并发控制
- **双路混合检索**：Dense Vector (COSINE) + Sparse BM25 (TF-IDF) → RRF 融合 → LLM Rerank
- **完整测试覆盖**：25+ 单元测试 + 3 个端到端测试脚本 + 评测框架
